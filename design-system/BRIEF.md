# Claude Design 작업 브리프 — 논리퍼즐 수업 슬라이드 디자인 시스템

너는 이 프로젝트(`논리퍼즐 수업 디자인 시스템`)의 디자인 시스템을 완성한다.
산출물은 reveal.js 슬라이드 덱에 적용될 **디자인 토큰 + 컴포넌트 스타일**이다.
아래 "계약(파일·클래스 이름)"은 **반드시 유지**하라 — 헤드리스 라즈베리파이의
`build.py`가 이 이름들에 의존하며, 이름을 지키면 1~32주 모든 덱이 자동 재빌드된다.
**이름은 유지, 값(색·타이포·여백·마크업 디테일)만 너의 디자인으로 교체**하라.

## 프로젝트 목적
중학생 5명 소그룹, 주 1회 90분, 교실 프로젝터. 흥미로운 논리퍼즐로 토론을 유발해
방정식·함수·그래프·선형대수·확률·미적분 같은 수학 개념으로 발전시키는 능동적 토론 수업.
디자인의 목적은 장식이 아니라 **수학적 사고의 시각화**다. 한 슬라이드 = 한 생각.
추론의 흐름(가정→모순→결론)이 화면에서 눈으로 읽혀야 한다. 모든 텍스트 한국어.
톤: '놀라움/아하!'를 주되 중학생에게 유치하지 않은 절제된 세련됨.

## 32주 = 4단계 컬러 코딩 (tokens.css의 body[data-phase])
- Phase 1 (1~8주) 논리와 추론      → 인디고/블루
- Phase 2 (9~16주) 변수와 방정식    → 틸/그린
- Phase 3 (17~24주) 함수와 그래프   → 앰버/오렌지
- Phase 4 (25~32주) 고급 개념의 씨앗 → 바이올렛/퍼플
베이스(배경·본문·중립)는 4단계 공통 유지, `--phase-accent`만 바뀐다.

## 의미 기반 색상 시스템 (1~32주 불변의 시각 문법)
색을 의미로 쓰고, 색맹 대응을 위해 항상 아이콘/형태와 페어링하라.
- 가정 `--c-assumption` □   · 모순 `--c-contra` ⚡
- 결론/정답 `--c-conclude` ✓ · 함정/오답 `--c-trap` ⚠
- 개념 등장(리빌) → `--phase-accent` 스포트라이트

## 계약 ① tokens.css — 이 CSS 변수 이름을 유지하라
--phase-accent, --phase-accent-soft (body[data-phase="1".."4"]로 전환)
--c-assumption, --c-contra, --c-conclude, --c-trap
--bg, --bg-soft, --ink, --ink-soft, --line
--font-sans, --font-mono
공통 클래스: .slide-band(상단 Phase 색 띠), .kicker(상단 라벨),
.tag.assume/.contra/.conclude/.trap

## 계약 ② components/components.css — 이 클래스 이름을 유지하라
- `table.truth` (+ `td.ok`, `td.no`, `tr.win`) — 진리표 / 경우 검사 표
- `.proof-flow .step` (+ `.step.contra`, `.arrow`) — 가정→모순 흐름도
- `ol.reason` — 번호 매긴 추론 단계
- `.trapbox` — 함정/흔한 오답 박스
- `.reveal-box .concept` — 개념 등장(리빌) 스포트라이트
추가로 만들면 좋은 컴포넌트(이름은 자유, 단 문서화):
좌표평면·그래프(시침/분침 교점, 약속시간 넓이확률), 상태공간 노드-엣지(강 건너기),
0/1 격자·벡터/내적(mod 2, 가중치), 수직선.

## 슬라이드 아키타입 (reveal.js `<section>` 단위, 11종)
타이틀 / 워밍업·함정 / 무대 설정 / 문제 / 풀이 구분 / 풀이(단계) /
개념 연결(리빌·가장 중요) / 증명 사례(가정→모순→∎) / 연습·확장 과제 /
표·진리표 / 끝. 각 아키타입의 reveal.js 마크업 패턴(권장 클래스 포함)을 제시하라.

## 타깃·제약
- reveal.js 5.x. 슬라이드는 `<section>`, 발표자 노트는 `<aside class="notes">`.
- 16:9. 교실 맨 뒤에서도 읽히는 큰 본문, 명도 대비 WCAG AA 이상.
- 한국어 폰트 페어링(예: Pretendard). 웹폰트를 쓰면 오프라인 단일 파일에
  base64로 인라인 가능하도록 별도 파일로 제공하라.
- 결과물은 자기완결형(오프라인 더블클릭)으로도 열려야 하므로 외부 CDN fetch 금지.

## 산출물
1. `tokens.css` (계약 ① 유지, 값 교체)
2. `components/components.css` (계약 ② 유지/확장)
3. 슬라이드 아키타입 11종의 마크업 예시 + 미리보기 카드(@dsCard)
4. 각 디자인 결정에 '교육적 이유' 한 줄
